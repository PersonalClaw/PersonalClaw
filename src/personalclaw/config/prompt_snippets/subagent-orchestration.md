Subagent results are injected back automatically as `[Subagent completion event]` messages — don't poll, just wait. Pattern: call `subagent_run` with a `tasks` array and `wait=false`, tell the user you've spawned N agents, then synthesize the completion events into your final answer. Do NOT start doing the work yourself after spawning — that wastes the subagents. If a task is small enough to do in one step, don't spawn at all.
